关于创建个人博客教程

2024-07-01
教程

在Github Page部署免费、好用的博客

新手任务开始前

请先在计算机上安装Git,Node.js,Github 账号

在计算机上创建hexo文件夹，在文件夹中右键打开 “Open Git Bash here”
在弹出窗口中，依次输入下面三行命令
npm install hexo-cli -g
hexo init blog
cd blog
打开在hexo文件夹中生成的blog文件夹，在blog文件夹中右键打开 “Open Git Bash here”
在弹出窗口中，依次再输入下面两行命令
npm install
hexo s
这里使用的是由Shen-Yu制作的ayer样式
在blog文件夹中右键打开 “Open Git Bash here” （也可以输入刚刚输入两行命令的窗口）
在弹出窗口中，依次再输入下面命令
git clone https://github.com/Shen-Yu/hexo-theme-ayer.git themes/ayer
将博客根目录下的 _config.yml 里的 theme 值修改成 ayer
如果需要更新（新下载不需要），再输入ayer更新命令两条
cd themes/ayer
git pull
在随便那个（都行）文件夹中右键打开 “Open Git Bash here”
git config –global user.email “你的邮箱”
git config –global user.name “你Github的名字”
ssh-keygen -t rsa -C “你的邮箱”
然后输入三次回车，在输入以下命令
cat ~/.ssh/id_rsa.pub
复制出现的显示密钥，打开Github，把密钥加入个人SSH公钥。
加入后，再次打开原来的窗口，输入：
ssh -T git@github.com
等待出现successful（成功的）字样
在blog文件夹中右键打开 “Open Git Bash here”
打开_config.yml，将文件最下面“deploy: ”替换成

在blog文件夹中右键打开 “Open Git Bash here”
在弹出窗口中，依次再输入下面命令
npm install hexo-deployer-git –save
hexo g
hexo s
hexo d

创建成功后的常用命令

建议在部署在Github前写好个人博客

hexo g 的意思简单来说就是是生成网页
hexo s 就是本地运行自己的博客，可以在浏览器中查看
hexo d 部署博客到Github

如果你想写文章，在blog文件夹中右键打开 “Open Git Bash here”，输入：
hexo new “你的文章名字”

我的第一篇博客：欢迎来到新博客网站！

今天，我要分享一个特别的喜悦——我终于拥有了自己的博客！这不仅仅是一个简单的Github Page网站，它更是我心中一个充满希望和梦想的角落。

追忆往事，吾常常在各个平台上浏览着别人的博客，欣赏他们用文字记录生活、分享知识的文章。那时，我就在心里默默许下了一个愿望：我也要有一个属于自己的博客。

于是，我开始学习如何创建我的博客。我希望它不仅仅是一个展示我的地方，更是一个能够与他人交流、碰撞思想的平台。我希望通过我的博客，能够与更多志同道合的朋友，共同探索这个五彩斑斓的世界。

在我的创建新博客之前，我开始将我之前创建的旧的“LIB临时工作室”官方网站的网址，换成到更符合的网址。将这里做成我的博客。这个过程虽然充满了挑战，但每一步都让我感到无比充实和满足。我学习了如何设置博客的版面、如何发布文章……每一个细节都凝聚了我的心血和汗水。

在这个过程中，我要特别感谢那些给予我帮助和支持的人（尤其是教学UP主）。他们分享他们的经验，让我能够更快地成长和进步。没有他们的帮助，我的博客可能还只是一个简单的网页。

现在，我的博客已经正式上线了！我迫不及待地想要分享我的喜悦和成果。在未来的日子里，我会在博客上分享我的生活感悟、学习心得以及任何我觉得值得分享的事物。我也期待着能够收到来自世界各地的朋友们的留言和评论，让我们一起交流思想、分享快乐。

最后，我想对每一个观看我的博客的人说一声谢谢。是你们的支持和鼓励让我有动力继续写下去。我会珍惜这个与你们交流的机会，努力让我的博客变得更加精彩、更加有价值。让我们一起见证彼此的成长和进步吧！
